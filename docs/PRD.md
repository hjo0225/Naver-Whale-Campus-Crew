# PRD — 네이버웨일 캠퍼스크루 부스 게임

## 1. 배경 / 목적

- 한양대 축제에서 네이버웨일 홍보 부스 운영
- 방문자가 웨일 모바일 앱에 추천인 코드를 등록하면 부스에서 카드 게임 참여
- 목표 지표: (예) 코드 등록 수, 부스 체류 시간, 만족도

## 2. 사용자

- **방문자(손님)** — 한양대 학생 / 축제 일반 방문객
- **운영진(캠퍼스크루)** — 부스 진행, 코드 발급/검증, 상품 지급

## 3. 사용자 흐름 (Happy Path)

1. 부스 방문 → 추천인 코드 안내
2. 웨일 앱 설치 / 실행 → 설정 → 추천인 코드 입력
3. 운영진이 등록 확인 → 부스 단말에서 게임 시작
4. NPC와 카드 게임(다회차) → 최종 점수 산출
5. 결과에 따른 상품 지급 (참가 시 키캡 또는 인형 택1, 1등 시 둘 다)

## 4. 핵심 기능

### 4.1 랜딩 / 홍보 슬라이드
- 자동 회전 슬라이드: 사이드바 / 웨일 스페이스 / 웨일온 / 그린드랍 / 멀티플레이

### 4.2 참가 안내 (3단계)
- STEP 1: 웨일 앱 추천인 코드 메뉴 안내
- STEP 2: 코드 복사 + 앱 다운로드 링크
- STEP 3: 부스 방문 + 상품 안내

### 4.3 게임 룰 안내 (5페이지)
- RULE 01: 카드 종류 (1~5 + 라마)
- RULE 02: 매칭 규칙 (같은 숫자 / +1 / 5↔라마)
- RULE 03: 차례 행동 (내기 / 뽑기 / 그만하기)
- RULE 04: 점수 계산 (중복 1회만, 라마 −8)
- RULE 05: 승리 조건

### 4.4 카드 게임 (멀티 라운드)
- NPC 2명과 대전 (별이, 페포 등 웨일프렌즈 캐릭터)
- 라운드별 점수 누적, 최종 1등 결정

### 4.5 결과 / 시상
- 라운드 결과 → 최종 결과 → 등수 및 상품 안내

## 5. 비기능 요구사항

- **오프라인 작동**: 부스 네트워크 불안정 가능 → 폰트 외 외부 의존성 최소화
- **단일 단말 운영**: 태블릿 / 노트북 풀스크린 가정
- **운영진 단축키**: 1/2/3/4/Enter/Esc — 화면에 노출되지 않음

## 6. 수용 기준 (Acceptance Criteria)

- [ ] 추천인 코드는 운영진이 손쉽게 교체 가능 (한 곳 수정으로 반영)
- [ ] NPC 턴 진행이 막힘 없이 돌아간다 (무한루프 방지)
- [ ] 라마 카드 매칭/점수 규칙이 룰 페이지와 코드에서 일치
- [ ] 라운드 종료/최종 결과 화면에서 점수 산출이 검증된다

## 7. Out of Scope

- SSR (단일 부스 PC, SEO 불필요)
- 멀티플레이어 (NPC 2명만)
- 회원가입 / 로그인
- DB / 게임 결과 영속화
- 중복 참가 방지 (운영진 수기 체크)
- 모바일에서의 게임 플레이 (랜딩 / 코드 안내만 모바일 대응)
