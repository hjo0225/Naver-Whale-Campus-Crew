# 부스 운영진 매뉴얼

축제 부스에서 본 웹앱을 운영할 네이버 서포터즈를 위한 안내. 손님이 직접 모바일로 플레이하지 않고, 부스 PC에서 운영진이 클릭하며 손님은 구두로 카드를 선택한다.

## 1. 사전 셋업 (부스 시작 전)

### 1.1 도메인 / 단말 준비
- 부스 PC 브라우저로 배포된 URL 접속 (`*.web.app`)
- F11로 전체화면. 게임 화면은 100vh fit 디자인이라 스크롤 불필요.
- 1366×768 이상 해상도 권장.

### 1.2 추천인 코드 교체
실제 부스 운영용 코드로 바꿔야 한다.
- 파일: `src/lib/config.ts`
- 한 줄: `export const REFERRAL_CODE = "ABCD1234";`
- 변경 후 `pnpm build` → `pnpm deploy` (또는 main에 push 시 GitHub Actions가 자동 배포).

### 1.3 상품 준비
- 키캡 / 인형 두 종류
- 두 판 모두 승 → 둘 다 증정
- 한 판 승 또는 0승 → 1개 택1

## 2. 손님 흐름

### 2.1 안내 (홈 / 슬라이드쇼)
- 손님이 부스에 도착하면 슬라이드쇼 자동 회전 중. 그대로 보여주거나 키보드 단축키로 빠르게 룰로 이동.
- 키보드 단축키 (운영진 전용, UI 표시 없음 — 추후 추가 예정):
  | 키 | 동작 |
  |---|---|
  | `1` / `H` | 홈 |
  | `2` | 참가 방법 |
  | `3` | 차별점 (랜딩 슬라이드 2) |
  | `4` | 게임 룰 |
  | `Enter` | 게임 룰 → 게임 |
  | `Esc` | 홈 (게임 중엔 비활성) |

### 2.2 추천인 코드 안내 (`/conditions/`)
- 손님 휴대폰을 받아 웨일 앱 설치 → 설정 → 추천인 코드 입력 안내.
- 코드는 화면에 표시 + 복사 버튼.

### 2.3 룰 설명 (`/rules/`)
- 30초 안에 핵심만:
  1. "1~5 카드와 라마. 손에 남은 만큼 점수 깎임 (낮을수록 승)."
  2. "같은 숫자 또는 +1만 낼 수 있어요. 5 위엔 라마, 라마 위엔 1."
  3. "내 차례엔 내기 / 뽑기 / 그만 셋 중 하나. 첫 턴은 무조건 내기."
  4. "두 판 — HYLION이랑 한 판, 네이버웨일이랑 한 판. 둘 다 이기면 키캡+인형, 한 판이라도 잡으면 1개."

### 2.4 게임 진행 (`/game/`)
- 손님이 구두로 "두 번째 카드 낼게요" 같이 지시 → 운영진이 해당 카드 클릭.
- 손님 첫 턴엔 그만하기/뽑기 버튼이 자동 잠겨 있어 헷갈릴 일 없음.
- NPC는 1.1초 사고 시간 후 자동 진행.
- 결과 화면 → "다음 상대 ▶" → R2.

### 2.5 결과 / 시상 (`/result/`)
- "완전 정복!" → 둘 다 증정
- "한 판은 잡았다!" → 키캡/인형 1개 택1
- "다음에 또 도전해주세요!" → 참가 상품 1개 (운영 정책 따라)
- "한 판 더" 버튼은 같은 손님이 한 번 더 하고 싶을 때만. 보통 다음 손님으로 넘긴다.

## 3. 운영진 멘트 통일 (조작 의혹 차단)

R1(HYLION)은 의도적으로 NPC 난이도가 낮게 세팅되어 있다. **이 사실은 손님에게 절대 노출하지 않는다.** 다음 표현으로 통일한다.

- 손님 질문: "왜 첫 판이 더 쉬워요?"
  → "HYLION은 워밍업 상대고, 네이버웨일이 진짜 승부예요!"
- 손님 질문: "왜 두 번째가 더 어렵죠?"
  → "두 번째는 본 게임이라 그래요. 한 판은 잡으셨으니 충분합니다!"

## 4. 자주 묻는 질문

| 질문 | 답변 |
|---|---|
| 모바일에서도 할 수 있어요? | 게임은 부스 PC에서만 가능합니다. 모바일은 안내 / 코드 등록만 |
| 동점이면 어떻게 되나요? | 동점은 손님 승으로 인정합니다 |
| 한 판만 이겨도 상품 받나요? | 네, 키캡 또는 인형 중 1개 받으실 수 있습니다 |
| 게임 중에 멈출 수 있나요? | "그만하기" 버튼으로 라운드 이탈 가능 (단 첫 턴은 불가) |
| 라마가 뭐예요? | -8점짜리 폭탄 카드입니다. 빨리 던지는 게 좋아요 |

## 5. 트러블슈팅

| 증상 | 조치 |
|---|---|
| 게임이 멈췄다 (NPC 차례에서 진행 X) | F5로 새로고침. 라운드부터 재시작 |
| 화면이 잘림 | F11로 전체화면, 또는 브라우저 줌 100% 확인 |
| 인터넷 끊김 | Pretendard 웹폰트만 영향. 게임 자체는 그대로 작동 |
| 코드 복사가 안 됨 | 손님이 직접 손으로 입력하시도록 안내 (대문자 8자) |

## 6. 부스 마감

- 본 앱은 데이터를 저장하지 않음. 마감 시 단순히 브라우저 닫고 PC 종료.
- 코드 등록 수 / 게임 플레이 수는 운영진 수기 카운트.
