# Game Rules

부스 카드게임 룰의 단일 진실 공급원. 코드(`src/lib/game/`)와 본 문서가 어긋나면 **본 문서를 먼저** 갱신한 뒤 코드를 고친다.

> 2026-04-30 변경: 1대1 두 라운드 → **4인 한 판** 구조로 전환. 결정 배경은 ADR-0003 참고.

## 1. 카드 구성

| ID | 이름 | 캐릭터 키 | points |
|---:|---|---|---:|
| 1 | 별이 | `byul_e` | 1 |
| 2 | 달토 | `dalto` | 2 |
| 3 | 페포 | `pepo` | 3 |
| 4 | 고스트웨일 | `ghost_whale` | 4 |
| 5 | 네이버웨일 | `naver_whale` | 5 |
| L | 라마 | (없음) | 8 |

**덱 구성**: 1~5 각 6장 + 라마 6장 = **총 36장**. 한 판 시작 시 새로 셔플.

**핸드**: 4명(손님 + NPC 3) 모두 **4장**으로 시작. 1장은 바닥(top)에 깔린다. 덱 잔여 = 36 − 16(분배) − 1(top) = **19장**.

## 2. 매칭 규칙 (`canPlay`)

(불변)

바닥 카드가 `top`일 때, 플레이어 카드 `card`를 낼 수 있는 조건:

1. `card.value === top.value` — **같은 값**
2. `top`과 `card` 모두 숫자이고 `card.value === top.value + 1` — **+1**
3. `top.value === 5 && card.value === 'LLAMA'` — **5 위에 라마**
4. `top.value === 'LLAMA' && card.value === 1` — **라마 위에 1**

그 외는 모두 불가. `top`이 없으면(첫 수) 아무 카드나 낼 수 있다.

## 3. 차례 행동 (3택 1)

(불변)

- **카드 내기** — 매칭되는 카드를 골라 바닥에 둔다. 손에 카드가 0이 되면 라운드 즉시 종료.
- **카드 뽑기** — 덱에서 1장 가져오고 차례 종료. 덱이 비었으면 자동으로 그만하기.
- **그만하기** — 이 라운드 이탈. 손에 든 카드로 점수 확정, 더 이상 차례가 돌아오지 않는다.

**제약**: 활성 플레이어가 자기 혼자 남은 상황에서는 **뽑기 불가**. 낼 카드도 없으면 라운드 즉시 종료.

## 4. 라운드 종료 조건 (`checkRoundEnd`)

(불변)

다음 중 하나라도 참이면 라운드 종료:

- 누군가의 손이 0이 되었다 (그 즉시).
- 활성 플레이어(quitted = false)가 0명.
- 활성 플레이어가 1명만 남았고, 그 사람이 낼 수 있는 카드도 없다 (`canPlay` 모두 false).

## 5. 점수 / 등수 / 상품

### 점수 계산 (`calculateScore`) — 불변
라운드 종료 시 손에 남은 카드들의 `points` 합산. 단:
- **같은 `value`의 카드는 한 번만 카운트한다** (의도된 룰). 예: `4, 4, 4` → 4점만, `1, 4, 4, L` → 1 + 4 + 8 = 13점.
- 손이 비어 있으면 0점.
- 총 점수는 음수 표기 (`formatScore`: `0` 또는 `-N`).

### 등수 (4인전 신규)
손님 등수 = **(손님보다 엄격히 점수 낮은 NPC 수) + 1**.

**동점은 손님이 더 높은 등수** (부스 우호 정책). 즉 NPC와 같은 점수면 NPC가 손님 아래로 밀린다.

### 상품 매핑

| 손님 등수 | `PlayerSummary.prize` | 상품 |
|---:|---|---|
| 1등 | `"both"` | 키캡 + 인형 둘 다 |
| 2·3등 | `"one"` | 키캡 또는 인형 중 1개 택1 |
| 4등(꼴등) | `"cheer"` | 응원품 |

## 6. NPC 구성

`CONFIG.opponents` (길이 = 3, 한 판의 NPC):

| 이름 | 캐릭터 |
|---|---|
| 달토 | dalto |
| 페포 | pepo |
| 웨일 | naver_whale |

NPC 3명은 모두 동일한 휴리스틱(§8)을 사용한다. 난이도 분기는 없다.

부스 우호 의도: 손님 1·2·3등 비율 ≥ 75%. D-day 전 시뮬레이션 검증 필요 (`src/lib/game/__tests__/simulation.test.ts` 갱신 후 `describe.skip` 해제).

## 7. 첫 턴 강제 플레이

(불변) 손님이 라운드 첫 턴이면 그만하기 / 뽑기는 잠금. 낼 카드 없을 때만 뽑기 허용. UI에서만 강제 (룰 자체는 동일).

## 8. NPC AI 휴리스틱 (`decideNpcMove`)

`decideNpcMove(npc, state)` — 모든 NPC가 동일 로직.

1. 낼 수 있는 카드가 1장 이상이면 → **points가 가장 큰 카드부터** 낸다 (라마 우선 폐기).
2. 낼 카드가 없고 혼자 남은 상태면 → 의사결정 없음.
3. 그 외 — **손패·점수 임계 + 확률 quit**:
   - 덱이 비었음 → **그만하기** (강제)
   - 손패 ≤ `QUIT_HAND_THRESHOLD`(=5) AND 점수 ≤ `QUIT_SCORE_THRESHOLD`(=10)
     AND `Math.random() < QUIT_PROBABILITY`(=0.3) → **그만하기**
   - 그 외 → **뽑기**

부스 우호 설계: 손패가 크거나(>5장) 손실이 큰(점수 >10) NPC는 절대 quit 하지 않는다. 손패가 적고 손실도 -10 이하인 NPC만 매 턴 30% 확률로 quit. 큰 손실을 굳히는 quit를 차단해 NPC가 가능한 한 더 뽑게 만든다(NPC 점수 ↑ → 손님 등수 유리).

NPC 사고 딜레이는 `CONFIG.npcThinkDelay` ms (현재 2200 — 부스에서 카드 흐름을 또렷이 보이려고 1/2 속도). UI 모션도 같은 톤으로 spring stiffness 절반.
